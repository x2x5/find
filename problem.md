# 清理按钮 + 输入框宽度调整失败

## 需求
1. 输入框左边加一个清空按钮，左边缘与论文面板左边缘对齐
2. 输入框宽度相应缩短

## 错误现象
1. 清理按钮不可见（未显示）
2. 输入框被意外收窄（355px → 260px）

## 根本原因

### 1. 清理按钮不可见
- 按钮 `display:none` 初始隐藏，依赖 JS 根据输入框内容切换显示
- `position: absolute` + `left: 379px` 虽然使搜索容器定位正确，但按钮本身的显示/隐藏逻辑依赖 `input` 事件触发
- 用户在没有输入内容时看不到按钮 → 以为是缺失

### 2. 输入框被收窄
- 为给清理按钮腾出空间（按钮 30px + gap 8px），将输入框从 355px 缩小到 260px（减少了 95px）
- 但用户没有要求改变输入框宽度，只要求加按钮
- 应该保持输入框宽度不变，通过调整整体搜索区域的左右位置来容纳按钮

### 3. 一次性改动过多
- 同时改了 HTML（加按钮）、CSS（定位方式、输入框宽度）、JS（显示/隐藏逻辑）
- 多个变化叠加，相互影响，难以单独验证每一处

## 教训
- 加按钮时不应压缩已有元素的尺寸，应通过调整容器位置来容纳
- 一次性只做一个改动，验证通过后再做下一个
